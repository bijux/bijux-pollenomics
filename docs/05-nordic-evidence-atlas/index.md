# Nordic Evidence Atlas

The Nordic Evidence Atlas is the shared Nordic map publication.

Open this page first when the fastest path is to inspect what readers actually
see before reviewing source-specific tables, normalized outputs, or package
internals.

The atlas is the main public evidence surface of the repository. This page
routes from a visible layer, point, or polygon to the exact source, output,
fieldwork, or runtime explanation that can support or limit what it appears to
show.

## Start Here

- open the atlas first when you need to inspect what a reader actually sees on
  the publication surface
- open [atlas output reference](https://bijux.io/bijux-pollenomics/02-bijux-pollenomics-data/outputs/nordic-atlas/)
  when the question is about generated files, checked-in assets, or publication
  packaging
- open [data reference](https://bijux.io/bijux-pollenomics/02-bijux-pollenomics-data/sources/) when the real
  question is where a visible layer or record originates
- open [fieldwork](https://bijux.io/bijux-pollenomics/04-fieldwork/) when a point appears to represent a
  documented visit and you need direct local context
- open [runtime handbook](https://bijux.io/bijux-pollenomics/01-bijux-pollenomics/) when the question is how
  the atlas was rebuilt, validated, or packaged

## Open This Page When

- opening the current atlas quickly without navigating through output reference
  pages first
- checking the publication surface that readers will actually inspect
- branching into data, fieldwork, or package docs once a concrete atlas
  question appears

## Open Another Page When

- explaining every upstream data caveat in detail
- documenting rebuild commands or release automation
- replacing the output-reference pages that describe the checked-in atlas files
- assuming the map alone is sufficient evidence without checking source or
  fieldwork context

## Bottom Line

Open this page when the visible map is the fastest path into the evidence
surface. A trustworthy read usually starts here and then branches into source,
output, fieldwork, or runtime docs depending on what the visible map element
is actually claiming.
