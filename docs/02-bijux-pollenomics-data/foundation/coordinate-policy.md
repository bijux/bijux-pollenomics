# Coordinate Policy

Spatial outputs should be normalized so atlas rendering and country filtering
can rely on one consistent interpretation of location.

## Current Policy

- preserve source identity while reshaping coordinates into repository-friendly
  outputs
- use normalized context layers for publication-facing rendering
- keep boundary-compatible country filtering central to the data workflow

## Purpose

This page records the repository stance on coordinate normalization and spatial
consistency.
