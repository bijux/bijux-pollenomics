# Projects

Projects are the intake boundary, not the published geography. A project tells
the repository which archive accession is in scope, which paper belongs to it,
and whether the supporting supplement has been captured well enough to begin
sample extraction.

## What A Project Row Settles

- which archive accession the repository is curating
- which paper DOI is treated as the project anchor
- whether supplementary material is required and archived
- whether the project has progressed into sample, site, and chronology tables

The project surface is where database breadth starts. It is deliberately wider
than the Nordic atlas because many tracked projects remain blocked before any
map-ready point can be justified.
