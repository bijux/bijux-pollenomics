# Release Support

Release support is where the repository decides whether public wording is ahead
of public evidence.

## Current Maintainer Checks

- runtime and report tests
- repository truth, output-governance, and docs contract checks
- animal output audits and readiness surfaces
- package and badge synchronization checks

## Release-Facing Review Order

1. read the [repository product model](../../report/repository_product_model.md)
2. read the [repository credibility dashboard](../../report/repository_credibility_dashboard.md)
3. read the [repository brutal honesty review](../../report/repository_brutal_honesty_review.md)
4. stop at the [repository final release refusal](../../report/repository_final_release_refusal.md)

If the refusal surface still blocks final release language, the repository is
not allowed to talk around that block with smoother prose.
