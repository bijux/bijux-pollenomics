# Operating Guidelines

Maintainer-package changes should clarify repository contracts, not obscure
them.

## Guidelines

- keep runtime and maintenance behavior separate
- prefer checks that fail early and clearly
- document new repository policy when maintainer tooling changes

## Purpose

This page records how `bijux-pollenomics-dev` should be evolved.
